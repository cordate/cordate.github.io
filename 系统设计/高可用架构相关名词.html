高可用架构相关名词

发表于 2019-08-22 更新于 2019-08-24 分类于 系统设计 ， 微服务 阅读次数：
本文字数： 4.7k 阅读时长 ≈ 4 分钟

高可用架构中有很多的名词和说明，理解这些名词和说明有利于大家学习交流，特别是对于软件架构师会有很多的帮助。这里讲的某大厂的架构。

物理网络相关

缩写 全称 解释
BGP Border Getaway Protocol BGP（边界网关协议）是一种基于TCP协议的动态路由协议，主要应用于不同自治域间交换路由信息和网络可达信息
ABTN Alibaba Backbone Transmission Network 阿里巴巴骨干传输网，通过BGP与全球ISP互联互通，用于承载互联网用户访问IDC应用流量
ACTN Alibaba Core Transmission Network 阿里巴巴核心传输网，用于承载DC之间的内网交互流量
AOTN Alibaba Optical Transmission Network 阿里巴巴光传输网，是底层L0网络，同时承载了ABTN和ACTN
ASW Access Switch 经典三层网络的接入层，也称tor（Top of Rack）
LVS Linux Virtual Server Linux虚拟服务器，这里泛指网关类负载均衡服务器。LVS是四层负载均衡，也就是说建立在OSI模型的第四层——传输层之上，传输层上有我们熟悉的TCP/UDP协议，LVS支持TCP/UDP的负载均衡。LVS的转发主要通过修改IP地址（NAT模式，分为源地址修改SNAT和目标地址修改DNAT）、修改目标MAC（DR模式）来实现, 可以参考学习LVS负载均衡集群
BSW Base Service Switch 基础服务交换机，给DNS/NTP/Syslog/AAA等基础服务server提供接入
LSW LVS Switch 负载均衡交换机，给SLB/XGW/LVS提供接入服务
MAN Metropolitan Area Network 域域网
MC MAN Core 域域网核心交换机
NAT Network Address Translation 网络地址转换
NSW NAT Switch NAT交换机，给ANAT（Alibaba NAT）提供接入服务
DCI Data Center Interconnection 数据中心互联
DSW DCI Switch 数据中心互联交换机，也是经典三层网络拓扑的核心层
PoD Points of Delivery 交付单元，指网络建设和交付的最小单元，一个PoD可以容纳的服务器规模定义为768台（16组ASW*48端口）
PSW PoD Switch 经典三层网络的汇聚层
A Record Address Record A记录，IPv4 Address record。A记录是用来指定主机名（或域名）对应的IPV4 IP地址记录
AAAA Record Quad-A Record 用来指定主机名（或域名）对应的IPv6 IP地址记录。
CNAME Canonical Name Record 别名记录，这种记录允许您将多个名字映射到另一个域名。
Underlay Underlay Network 传统的物理网络，区别于Overlay虚拟网络。通俗的讲，没有SDN和虚拟化之前，基础的数通网络。

虚拟网络相关

缩写 全称 解释
IDC Internet Data Center 机房，其中包含多个IDC网络架构，是网络架构模块化设计的最小单元
VPC Virtual Private Connection 虚拟专用网络
SLB Software Load Balancer VPC组件，负载均衡基础设施，为外部用户提供软件负载均衡服务
VIP Virtual IP Address 虚拟IP技术，IP技术的一种，增强网络管理，发挥出VLAN的优势，改变了网络结构，合理分配网络资源，均衡网络负载，有效降低网上广播信息，方便对用户的分组管理。通常指用户购买的公网IP地址，分为四层和七层VIP，存在于SLB上
ENI Elastic Network interface 弹性网卡

云基础服务/阿里云

缩写 全称 解释
ECS Elastic Compute Service 云服务器ECS是一种弹性可伸缩的计算服务
ISP Internet Service Provider 互联网服务提供商，电信运营商
VPC Virtual Private Cloud 专有网络VPC是构建在物理网络之上的虚拟化网络。是一个隔离的网络环境，并可以自定义IP地址范围、网段、路由表和网关等；此外，也可以通过专线/VPN/GRE等连接方式实现云上VPC与传统IDC的互联，构建混合云业务
OXS OXS区网络 公有云上的网络分为两大子网；就是传说中的OXS和售卖区。
HDR Hybird Disaster Recovery 混合云容灾服务。HDR是一个为数据中心提供企业级应用的本地备份与云上容灾一体化的服务

阿里基础设施/中间件

缩写 全称 解释
AServer AServer是一个（无线+pc）统一接入（Nginx/Tengine）网关（Aone应用），为阿里内部应用快速接入上线提供了高效稳定的环境保障，同时提供了证书自动续签、安全套件、多种调度策略支持，接入方只需极小的接入成本就可得到功能丰富、极速稳定的接入保障。
AMDC 阿里巴巴移动（无线）调度中心（Aone应用）
DNS Domain Name System 域名解析系统
Local DNS Recursive Name Server 也叫递归DNS，介于终端用户和权威DNS之间，提供递归查询DNS Tree和缓存域名记录的功能
ADNS Alibaba DNS 内部研发的高性能DNS
CDN Content Delivery Network 内容分发网络，将源站内容分发至最接近用户的节点，使用户可就近取得所需内容，提高用户访问的响应速率和成功率。解决因分布、带宽、服务器性能带来的访问延迟问题，适用于站点加速、点播、直播等场景
GSLB Global Server Load Balance 全局负载均衡。实现在广域网上不同地域的服务器间的流量调配，保证使用最佳的服务器服务离自己最近的客户，从而确保访问质量
AMDC Alibaba Mobile Dispatch Center 无线调度服务，为了对无线app进行更好的域名解析，以及进行业务解析工作。基于最基本的HTTP DNS解析，逐渐增加各种业务解析，演化成了现在的AMDC。
七网隔离 七网隔离 七网隔离
GOC Global Operations Center 全球运行指挥中心
GNS Global Name Server GNS是目前内部中间件依赖的元数据中心。存储数据主要是中间件及业务关心的在线元数据（如应用、机器的环境信息、统计数据，依赖关系等）
Overlay Overlay Network 覆盖网络，Overlay网络是将已有的物理网络（Underlay网络）作为基础，在其上建立叠加的逻辑网络，实现网络资源的虚拟化。Overlay是应用层网络，它是面向应用层的，不考虑或很少考虑网络层，物理层的问题。
SDN Software Defined Network 软件定义的网络，是由美国斯坦福大学CLean State课题研究组提出的一种新型网络创新架构，是网络虚拟化的一种实现方式。

常规的名词

缩写 全称 解释
GHA Global High Availability 全局高可用
LDR Local Disaster Recovery 同城容灾
RDR Remote Disaster Recovery 异地容灾
CDR Cloud Disaster Recovery 云灾难恢复
DRP Disaster Recovery Plan 灾难恢复预案
BIA Business Impact Analysis 商业影响分析，也称做业务影响评估
Failover Fail Over 故障转移，又称故障切换，一般指IDC应用出现故障时恢复应用的过程，即容灾恢复
Failback Fail Back 故障恢复，一般是指当IDC内的环境恢复后，将应用数据迁回自有IDC恢复应用运行的过程
RTO Recovery Time Objective 恢复时间目标。故障发生后，期望从启动容灾恢复操作到应用恢复上线所需要的时间，RTO是衡量服务中断的时间长度。故障单位时间内对业务造成的损失越大，RTO就要求越短
RPO Recovery Point Objective 数据恢复点目标。指应用发生故障时可以容忍的数据丢失量。指当服务恢复后，恢复到的数据所对应的时间点，主要取决于数据同步的延迟。数据越重要，RPO就要求越小。RPO越高，往往要求数据备份、复制频率更高，对生成环境、网络的压力也会越大，成本通常也越高。
SLA Service Level Agreement 服务等级协议。是关于网络服务供应商和客户间的一份合同，其中定义了服务类型、服务质量，以及在服务不达标情况下的服务赔偿。SLA是在一定开销下为保障服务的性能和可靠性，服务提供商与用户间定义的一种双方认可的协定。
SLI Service Level Indicator 服务等级指标。测试服务等级的各种基本指标，通常是经过量化了的
SLO Service Level Object 服务等级目标。指定了服务所提供功能的一种期望状态。实际上，SLA的保障是以一系列的服务等级目标（SLO）的形式定义的。服务等级目标是一个或多个有限定的服务组件的测试的组合。SLO是用SLI来描述的
基线 Base Line 基线是软件工程活动从一个环节转入另一个环节时对阶段产品或组件的标识。因为软件规模的膨胀和分工的细化，软件开发过程变得越来越复杂，每个阶段可能由不同类型的角色和人员来完成，因此有必要清晰标识上一阶段完成的成果和下阶段开始工作的基础。这种标识活动就是建立基线。
IOHang IO hang hang：卡主挂起。IO hang就是服务器磁盘读写过慢，导致线程和进程挂起，大量读写线程/进程挂起导致服务器宕机
QoS Quality of Service 服务质量，指一个网络能够利用各种基础技术，为指定的网络通信提供更好的服务能力，是网络的一种安全机制，是用来解决网络延迟和阻塞等问题的一种技术。
DSCP Differentiated Services Code Point 差分服务代码点，IETF于1998年12月发布了Differentiate Service的QoS分类标准。它在每个数据包IP头部的服务类别TOS标识字节中，利用已使用的6比特和未使用的2比特，通过编码值来区分优先级。其实DSCP就是为了保障通信的QoS，在数据包IP头部的8个标识字节进行编码，来划分服务类别，区分服务的优先级。
